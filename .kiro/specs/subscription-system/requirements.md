# Requirements Document: Subscription System

## Introduction

The Subscription System enables monetization of the exam library through a freemium model. Free users get 5 downloads per month, while premium users get unlimited downloads. Premium access is granted through one-time codes that students purchase by contacting the administrator directly.

## Key Design Decisions

1. **No User Authentication**: Users remain anonymous, subscription data stored locally on device
2. **Device-Based**: Each device gets its own subscription (no account sync across devices)
3. **Local Storage**: Subscription state stored in device storage (IndexedDB/AsyncStorage)
4. **Code Validation**: Codes validated against Supabase to prevent reuse
5. **Offline-First**: Download limits enforced locally, synced when online

## Glossary

- **Free_User**: User with default 5 downloads/month limit
- **Premium_User**: User with unlimited downloads after activating a code
- **Subscription_Code**: One-time use code generated by admin for premium access
- **Device_ID**: Unique identifier for the device (generated on first app launch)
- **Download_Quota**: Number of downloads remaining in current month
- **Monthly_Reset**: Automatic reset of download count on the 1st of each month

## Requirements

### Requirement 1: Device Identification

**User Story:** As the system, I need to uniquely identify each device, so that subscription data is tracked per device.

#### Acceptance Criteria

1. WHEN the app launches for the first time, THE system SHALL generate a unique Device_ID
2. THE Device_ID SHALL be stored in persistent local storage
3. THE Device_ID SHALL remain constant across app restarts
4. THE Device_ID SHALL be used to track subscription status locally
5. THE Device_ID SHALL be sent to Supabase when validating codes

### Requirement 2: Free Tier Download Limits

**User Story:** As a free user, I want to download up to 5 papers per month, so that I can access some content without payment.

#### Acceptance Criteria

1. WHEN a new device first uses the app, THE system SHALL initialize a free subscription with 5 downloads/month
2. THE system SHALL track downloads_this_month locally
3. WHEN a user downloads a paper, THE system SHALL increment downloads_this_month
4. WHEN downloads_this_month reaches 5, THE system SHALL prevent further downloads
5. THE system SHALL display remaining downloads (e.g., "3/5 downloads remaining")
6. ON the 1st of each month, THE system SHALL reset downloads_this_month to 0

### Requirement 3: Download Limit Notification

**User Story:** As a free user, I want to see my download limit when I try to download, so that I know when to upgrade.

#### Acceptance Criteria

1. WHEN a free user attempts to download a paper, THE system SHALL show a toast notification with remaining downloads
2. THE notification SHALL display: "Download started. X/5 downloads remaining this month"
3. WHEN downloads_this_month reaches 5, THE notification SHALL say: "Download limit reached (5/5). Upgrade to Premium for unlimited downloads"
4. THE notification SHALL auto-dismiss after 4 seconds

### Requirement 4: Premium Code Entry Dialog

**User Story:** As a free user who reached the limit, I want to enter a premium code, so that I can get unlimited downloads.

#### Acceptance Criteria

1. WHEN a free user attempts to download after reaching the limit, THE system SHALL show a dialog
2. THE dialog SHALL display: "Download Limit Reached" title
3. THE dialog SHALL show: "You've used all 5 free downloads this month"
4. THE dialog SHALL provide two options: "Enter Premium Code" and "Cancel"
5. WHEN user clicks "Enter Premium Code", THE system SHALL show a code input dialog
6. THE code input SHALL accept alphanumeric codes (format: XXXX-XXXX-XXXX)
7. THE dialog SHALL have "Activate" and "Cancel" buttons

### Requirement 5: Premium Code Validation

**User Story:** As a user entering a premium code, I want the code validated, so that only legitimate codes work.

#### Acceptance Criteria

1. WHEN user submits a code, THE system SHALL check internet connectivity
2. IF offline, THEN THE system SHALL show: "Internet required to activate premium code"
3. IF online, THE system SHALL query Supabase subscription_codes table
4. THE system SHALL check if code exists and is_used = false
5. IF code is invalid, THE system SHALL show: "Invalid code. Please check and try again"
6. IF code is already used, THE system SHALL show: "This code has already been used"
7. IF code is valid, THE system SHALL proceed to activation

### Requirement 6: Premium Activation

**User Story:** As a user with a valid code, I want premium activated immediately, so that I can download unlimited papers.

#### Acceptance Criteria

1. WHEN a valid code is confirmed, THE system SHALL update Supabase:
   - Set is_used = true
   - Set used_by = Device_ID
   - Set used_at = current timestamp
2. THE system SHALL update local subscription:
   - Set tier = 'premium'
   - Set max_downloads = -1 (unlimited)
   - Set activated_at = current timestamp
   - Set expires_at = current timestamp + duration_months
   - Set subscription_code = entered code
3. THE system SHALL show success message: "Premium activated! You now have unlimited downloads"
4. THE system SHALL allow the download to proceed immediately

### Requirement 7: Premium Status Display

**User Story:** As a premium user, I want to see my premium status, so that I know when it expires.

#### Acceptance Criteria

1. THE Library page SHALL display subscription status in top-left corner
2. FOR free users, THE display SHALL show: "Free: X/5 downloads"
3. FOR premium users, THE display SHALL show: "Premium: Unlimited" with a crown icon
4. WHEN user taps the status badge, THE system SHALL show a detail dialog
5. THE detail dialog SHALL show:
   - Subscription tier
   - Downloads this month (even for premium)
   - Expiration date (for premium)
   - Days remaining (for premium)
   - Option to "Enter Premium Code" (for free users)

### Requirement 8: Premium Expiration

**User Story:** As a premium user whose subscription expired, I want to be notified and downgraded gracefully, so that I can renew or continue with free tier.

#### Acceptance Criteria

1. THE system SHALL check expiration on app launch and before each download
2. WHEN current date > expires_at, THE system SHALL:
   - Set tier = 'free'
   - Set max_downloads = 5
   - Set status = 'expired'
   - Reset downloads_this_month = 0
3. THE system SHALL show a dialog: "Premium Expired"
4. THE dialog SHALL display: "Your premium subscription has expired. You can renew with a new code or continue with 5 free downloads per month"
5. THE dialog SHALL have options: "Enter New Code" and "Continue with Free"

### Requirement 9: Monthly Download Reset

**User Story:** As a free user, I want my download count reset each month, so that I get fresh downloads.

#### Acceptance Criteria

1. THE system SHALL check if month changed on app launch
2. THE system SHALL compare last_reset_date with current date
3. IF month or year changed, THE system SHALL:
   - Set downloads_this_month = 0
   - Set last_reset_date = current date
4. THE reset SHALL apply to both free and premium users (for tracking purposes)

### Requirement 10: Admin Code Generation

**User Story:** As an administrator, I want to generate premium codes, so that I can sell them to students.

#### Acceptance Criteria

1. THE Admin Panel SHALL have a "Subscription Codes" page
2. THE page SHALL have a "Generate Code" button
3. WHEN clicked, THE system SHALL show a form with:
   - Duration dropdown (1 month, 3 months, 6 months, 12 months)
   - Quantity input (how many codes to generate)
4. WHEN submitted, THE system SHALL generate codes with format: XXXX-XXXX-XXXX
5. THE codes SHALL be random, unique, and uppercase
6. THE system SHALL insert codes into Supabase subscription_codes table
7. THE system SHALL display generated codes for admin to copy

### Requirement 11: Admin Code Management

**User Story:** As an administrator, I want to view all codes and their status, so that I can track sales and usage.

#### Acceptance Criteria

1. THE Admin Panel SHALL display a table of all subscription codes
2. THE table SHALL show columns: Code, Duration, Status (Unused/Used), Used By, Used At, Created At
3. THE table SHALL support filtering by status (All/Unused/Used)
4. THE table SHALL support search by code
5. THE admin SHALL be able to copy codes to clipboard
6. THE admin SHALL see total counts: Total Codes, Unused Codes, Used Codes

### Requirement 12: Admin Subscription Analytics

**User Story:** As an administrator, I want to see subscription statistics, so that I understand revenue and user engagement.

#### Acceptance Criteria

1. THE Admin Dashboard SHALL display subscription metrics:
   - Total active premium users
   - Total codes generated
   - Total codes used
   - Revenue estimate (codes used × price)
2. THE Analytics page SHALL show:
   - Premium activations over time (chart)
   - Code usage by duration (pie chart)
   - Average downloads per user type
3. THE system SHALL track these metrics in Supabase analytics table

### Requirement 13: Offline Code Storage

**User Story:** As a user, I want my premium status to work offline, so that I can download papers without internet.

#### Acceptance Criteria

1. THE subscription data SHALL be stored in local device storage
2. THE system SHALL work offline for:
   - Checking download limits
   - Displaying subscription status
   - Allowing downloads (if quota available)
3. THE system SHALL require internet only for:
   - Validating new premium codes
   - Syncing download counts (optional)

### Requirement 14: Download Enforcement

**User Story:** As the system, I need to enforce download limits, so that free users cannot bypass restrictions.

#### Acceptance Criteria

1. BEFORE each download, THE system SHALL check:
   - Current subscription tier
   - Downloads this month
   - Max downloads allowed
2. IF tier = 'free' AND downloads_this_month >= 5, THE system SHALL block download
3. IF tier = 'premium' AND not expired, THE system SHALL allow download
4. THE system SHALL increment downloads_this_month after successful download
5. THE enforcement SHALL happen in downloadService before file download starts

### Requirement 15: Data Persistence

**User Story:** As a user, I want my subscription data saved, so that it persists across app restarts.

#### Acceptance Criteria

1. THE subscription data SHALL be stored in IndexedDB (web) or AsyncStorage (mobile)
2. THE data SHALL include:
   - device_id
   - tier
   - downloads_this_month
   - max_downloads
   - subscription_code
   - activated_at
   - expires_at
   - last_reset_date
3. THE data SHALL be loaded on app initialization
4. THE data SHALL be updated after each download and code activation

### Requirement 16: Error Handling

**User Story:** As a user, I want clear error messages, so that I understand what went wrong.

#### Acceptance Criteria

1. IF code validation fails due to network error, THE system SHALL show: "Network error. Please check your connection and try again"
2. IF code validation times out, THE system SHALL show: "Request timed out. Please try again"
3. IF Supabase is unreachable, THE system SHALL show: "Service temporarily unavailable. Please try again later"
4. ALL error messages SHALL be user-friendly and actionable

### Requirement 17: Localization

**User Story:** As a user, I want subscription messages in my language, so that I understand the system.

#### Acceptance Criteria

1. ALL subscription-related text SHALL support French and English
2. THE system SHALL use existing i18n system
3. Translations SHALL include:
   - Download limit messages
   - Premium activation messages
   - Error messages
   - Status labels

### Requirement 18: Security

**User Story:** As an administrator, I want codes to be secure, so that they cannot be guessed or reused.

#### Acceptance Criteria

1. THE codes SHALL be cryptographically random
2. THE codes SHALL be 12 characters (XXXX-XXXX-XXXX format)
3. THE codes SHALL use uppercase letters and numbers only
4. THE Supabase table SHALL have unique constraint on code column
5. THE code validation SHALL be atomic (prevent race conditions)
6. THE used codes SHALL never be reusable

### Requirement 19: Migration Strategy

**User Story:** As a developer, I need to migrate existing users, so that they get free tier by default.

#### Acceptance Criteria

1. WHEN the app updates with subscription system, THE system SHALL:
   - Generate Device_ID if not exists
   - Create free subscription with downloads_this_month = 0
   - Set last_reset_date = current date
2. EXISTING downloaded papers SHALL not count toward current month's limit
3. THE migration SHALL happen silently without user action

### Requirement 20: Testing Requirements

**User Story:** As a developer, I need to test the subscription system, so that it works reliably.

#### Acceptance Criteria

1. THE system SHALL have unit tests for:
   - Code generation
   - Code validation
   - Download limit enforcement
   - Monthly reset logic
   - Expiration checking
2. THE system SHALL have integration tests for:
   - Full premium activation flow
   - Download with limit enforcement
   - Expiration and downgrade
3. THE admin panel SHALL have a test mode to generate test codes
